# 对照品标准物质说明表生成

## 触发条件

当用户上传PDF文件且文件名包含以下关键词时，主动询问是否生成标准物质说明表：
- "COA"
- "Certificate"
- "Analysis"
- "检验报告"
- "批号"
- "Lot"
- "Batch"

## 询问话术

检测到用户上传了COA文件，主动询问：
"检测到COA文件，是否需要我帮你生成标准物质说明表？"

如果用户确认，继续执行生成流程。

## 执行流程

### 第一步：OCR读取COA内容

使用 JoinCare OCR API 读取COA PDF内容：
```bash
curl -X POST "https://ops.joincare.com/api/jky/ems/attach/ocr/file" \
  -H "Authorization: 852F88D8-E94A-44EB-A712-65CB167A1995" \
  -F "file=@COA.pdf"
```

提取以下关键字段：
| 字段 | 说明 |
|:---|:---|
| 药品名称 | Product Name / Description |
| 对照物质名称 | Substance Name / Chemical Name |
| 批号 | Lot No. / Batch No. |
| 生产厂家 | Manufacturer / Supplier (COA来源公司) |
| 英文名 | English Name / Synonyms |
| 有效期 | Expiration Date / Retest Date |
| CAS号 | CAS Number |
| 分子式 | Molecular Formula |
| 分子量 | Molecular Weight |
| 含量 | Assay / Purity (%) |
| 贮存条件 | Storage Condition |

### 第二步：验证信息

#### 2.1 验证公司信息
使用 web_search 工具查询：
- 公司名称是否正确
- 公司地址是否匹配
- 公司官网是否存在

**验证话术**：
"正在验证公司信息..."

#### 2.2 验证产品信息
使用 web_search 工具查询：
- CAS号是否正确
- 分子式是否匹配
- 分子量是否正确
- 产品名称是否标准

**验证话术**：
"正在验证产品信息..."

#### 2.3 批号格式检查
检查批号格式是否符合该供应商的编号规则。

### 第三步：发现问题并修正

验证完成后，向用户报告验证结果：

**报告格式**：
```
验证完成！以下是核查结果：

## ✅ 验证结果

### 公司信息
| 项目 | COA内容 | 验证结果 |
|:---|:---|:---|
| 公司名称 | XXX | ✅ 正确/⚠️ 需修正 |
| 地址 | XXX | ✅ 匹配 |

### 产品信息
| 项目 | COA内容 | 验证结果 |
|:---|:---|:---|
| CAS号 | XXX | ✅ 正确 |
| 分子式 | XXX | ✅ 正确 |
| 分子量 | XXX | ✅ 正确 |

## ⚠️ 发现的问题
1. [具体问题描述]

是否需要修正后重新生成？
```

### 第四步：修正问题

根据验证结果修正数据：
- 公司名称补充完整（如 Ltd. → Ltd.）
- 修正分子式格式（下标）
- 核对含量数值

**修正确认**：
"已修正：[具体问题]，重新生成说明表。"

### 第五步：生成Word文档

1. 打开模板文件 `assets/对照物质说明表模板.docx`
2. 按照格式规范填充数据
3. 保存生成的Word文档
4. 发送文件给用户

## 固定信息配置

以下信息在所有说明表中保持不变：
- **提供单位**：珠海保税区丽珠合成制药有限公司
- **经办人签名**：魏永红
- **联系方式**：13570680132

## 格式规范

生成的Word文档遵循以下格式：
- **中文字体**：宋体 小四（12pt）
- **西文字体**：Times New Roman 小四（12pt）
- **行距**：单倍行距
- **对齐方式**：垂直居中
- **分子式数字**：下标格式

| R8C2-4 | 使用方法（勾选"直接折算"） |
| R2C2-4 | 提供单位（固定：珠海保税区丽珠合成制药有限公司） |
| R3C2 | 经办人签名（固定：魏永红） |
| R3C4 | 联系方式（固定：13570680132） |

## 注意事项

- **来源（R3C2-4）**：填写经核实后的COA生产厂家完整名称
- **提供单位、经办人、联系方式**：使用固定信息
- **分子式**：数字必须设置为下标格式
- **复选框**：使用标准字符 □ 和 ☑
- **COA中无明确数据的字段**：保持空白，不擅自填写
- **验证发现问题时**：必须先报告用户，确认修正后再生成
