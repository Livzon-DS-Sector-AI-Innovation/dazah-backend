# CDE发补回复模板生成器 v2.0

根据CDE补充资料通知函和公司模板，自动生成符合公司格式规范的发补回复Word文档。

**v2.0更新：**
- 自动解析CDE PDF通知函，提取药品信息（名称、登记号、受理号等）
- 自动提取通知函中的所有问题（支持1/2/3级层级）
- 自动更新封面页三项内容（产品名称、登记号、受理号）
- 智能页眉处理，自动替换药品名称

## 触发条件

当用户执行以下操作时触发：
1. 上传CDE补充资料通知函（PDF）
2. 上传公司发补回复模板（Word）- 可选
3. 明确要求生成发补回复模板

## 使用流程

### Step 1: 接收用户文件

用户必须提供：
- **CDE补充资料通知函**（PDF格式）- 必需
- **公司发补回复模板**（Word格式）- 可选（如未提供使用默认格式）

### Step 2: 解析CDE通知函（自动）

使用Python脚本自动解析PDF：

**提取信息：**
- 药品名称（产品名称/原料药名称）
- 登记号
- 受理号
- 申请人/公司名称
- CDE通知日期和编号
- 所有补充问题（按层级结构：1、2、3... / （1）（2）（3）... / ① ② ③...）

**解析示例：**
```python
python3 scripts/generate_reply_template_v2.py \
  --notice "CDE通知函.pdf" \
  --template "公司模板.docx" \
  --output "输出模板.docx"
```

### Step 3: 生成文档

根据模板格式和通知函内容生成Word文档：

**文档结构：**
1. 封面页（自动填入解析的药品信息）
2. 目录页（根据提取的问题自动生成）
3. 正文（按问题层级排列）

**封面页自动填入内容：**
- 产品名称：[从PDF提取的药品名称]
- 登 记 号：[从PDF提取的登记号]
- 受 理 号：[从PDF提取的受理号]
- 申请人/注册代理机构名称：[从PDF提取的公司名称]

**每个问题的结构：**
```
（1）[问题原文，蓝色加粗 RGB(0,0,255)，从通知函提取]

【答复】[黑色加粗]
（请在此处填写回复内容）[黑色，首行缩进2字符]

【回复建议】[黑色加粗]
• 回复思路：[分析CDE关注点，提供回复策略][黑色]
• 回复模板：[标准回复格式][黑色]
• 补充实验：[是/否，如需补充列出具体实验][黑色]
• 实验项目：[具体实验项目列表][黑色]
• 法规依据：[引用ICH指南、药典等][黑色]
```

### Step 4: 格式验证

生成后必须检查：
- [x] 封面页三项内容已正确填入（产品名称、登记号、受理号）
- [x] 所有CDE问题均已收录
- [x] 问题编号连续无遗漏
- [x] 页眉格式与模板一致（含logo图片，药品名称已更新）
- [x] 字体、字号、颜色符合模板
- [x] 段落缩进、行间距正确

## 精确格式规范

### 封面页格式

| 内容 | 字号 | 格式 | 对齐 |
|------|------|------|------|
| **原料药登记资料** | **26pt** | 加粗 | 居中 |
| 资料类型、产品名称、登记号、受理号、申请人、申请事项、联系人、联系电话 | **16pt** | 常规 | 左对齐 |
| 注册地址、邮政编码 | **14pt** | 常规 | 左对齐 |
| 关联制剂受理号、Email | **16pt** | 常规 | 左对齐 |
| **[药品名称]** | **15pt** | 加粗 | 居中 |
| **补充资料研究报告** | **15pt** | 加粗 | 居中 |

**注意：封面页产品名称、登记号、受理号必须随品种修改！**

### 页眉格式（从目录页开始）

**布局：** 左侧文字（靠左）| 右侧logo+页码（靠右）

**三行文字（10.5pt）：**
- 原料药登记资料
- [药品名称]（无菌，原料药）
- 补充资料研究报告

**右侧：**
- 上部：公司logo图片
- 下部：第 X 页 共 Y 页

### 目录页格式

- **"目  录"**：14pt，加粗，居中
- 目录项：使用制表位右对齐（约15cm处），点线引导符
- 二级目录项缩进0.74cm
- **"附件目录"**：14pt，加粗，居中

### 正文格式

- **正文字号**：12pt（小四）
- **问题**：蓝色 RGB(0, 0, 255) 加粗
- **【答复】【回复建议】**：黑色加粗
- **行距**：1.5倍
- **首行缩进**：2字符（0.74cm）
- **字体**：中文宋体，西文Times New Roman

## 问题完整性验证

生成文档后，必须对照CDE通知函验证所有问题均已收录。

## 执行命令

```bash
python3 scripts/generate_reply_template_v2.py \
  --notice <CDE通知函PDF路径> \
  --template <公司模板Word路径> \
  --output <输出Word路径>
```

**参数说明：**
- `--notice`: CDE通知函PDF路径（必需）
- `--template`: 公司模板Word路径（可选）
- `--output`: 输出Word路径（必需）
- `--drug-name`: 药品名称（可选，默认从PDF提取）
- `--acceptance-no`: 受理号（可选，默认从PDF提取）
- `--registration-no`: 登记号（可选，默认从PDF提取）
- `--company`: 公司名称（可选，默认从PDF提取）

## 输出文件

生成Word文档，包含：
- 封面页（26pt标题，药品信息自动填入）
- 目录页（制表位对齐，点线引导符，根据提取的问题生成）
- 所有CDE问题的回复模板（原文+空白答复区+回复建议）
- 页眉（含公司logo图片和页码，药品名称已更新）

## 封面页XML修改参考

如需直接修改现有docx文件的封面页，可使用以下Python代码：

```python
import zipfile
import shutil

input_path = "输入.docx"
output_path = "输出.docx"
shutil.copy(input_path, output_path)

with zipfile.ZipFile(output_path, 'r') as zin:
    doc_xml = zin.read('word/document.xml').decode('utf-8')

# 修改产品名称（第一个出现）
doc_xml = doc_xml.replace('<w:t>原药品名</w:t>', '<w:t>新药品名</w:t>', 1)

# 修改登记号（第一个空格）
doc_xml = doc_xml.replace(
    '<w:t xml:space="preserve"> </w:t>',
    '<w:t>登记号</w:t>',
    1
)

# 修改受理号（第二个空格）
doc_xml = doc_xml.replace(
    '<w:t xml:space="preserve"> </w:t>',
    '<w:t>受理号</w:t>',
    1
)

with zipfile.ZipFile(output_path, 'a') as zout:
    zout.writestr('word/document.xml', doc_xml)
```
