### 基本信息

| **URL** | https://www.nmpa.gov.cn/yaopin/ypfgwj/index.html |
| **所属机构** | 国家药品监督管理局 (NMPA) |
| **内容类型** | 部门规章、规范性文件、工作文件 |
| **更新频率** | 每周 2-5 次 |
| **历史数据** | 27+ 页（约 500+ 条记录） |

### 可行性验证结果

| 验证项 | 结果 | 说明 |
|--------|------|------|
| ① 列表页可获取 | ⚠️ 需 Playwright + 反爬绕过 | httpx 返回 412 Precondition Failed |
| ② 标题可获取 | ⚠️ 需 Playwright + 反爬绕过 | 列表项格式: `标题(日期)` |
| ③ 发布日期可获取 | ⚠️ 需 Playwright + 反爬绕过 | 日期在标题括号中或独立字段 |
| ④ 详情链接可获取 | ⚠️ 需 Playwright + 反爬绕过 | 链接格式: `/yaopin/ypfgwj/20260527xxxx.html` |
| ⑤ 详情页可进入 | ⚠️ 需 Playwright + 反爬绕过 | 同样受瑞数6保护 |
| ⑥ 正文可获取 | ⚠️ 需 Playwright + 反爬绕过 | 详情页内容需渲染 |
| ⑦ 附件链接可获取 | ⚠️ 需 Playwright + 反爬绕过 | 附件在渲染后 DOM 中 |

### 技术方案

```
技术栈: Playwright + 反爬绕过 + BeautifulSoup
反爬策略: 瑞数6 (RS6) 动态安全 → 需要高级绕过方案
```

**详细方案**:

1. **反爬机制分析**:
   - NMPA 使用**瑞数6 (RS6)** 动态安全产品
   - 特征:
     - 动态代码混淆
     - 无限 debugger 陷阱
     - 请求参数动态签名
     - Cookie 动态生成
   - httpx 直接请求返回 412 Precondition Failed

2. **绕过方案（按优先级）**:

   **方案 A: Playwright stealth 模式**（推荐首选）
   ```python
   from playwright.async_api import async_playwright
   from playwright_stealth import stealth_async
   
   async def fetch_page(url):
       async with async_playwright() as p:
           browser = await p.chromium.launch(headless=True)
           context = await browser.new_context()
           page = await context.new_page()
           await stealth_async(page)  # 注入反检测脚本
           await page.goto(url)
           await page.wait_for_timeout(5000)  # 等待 JS 执行
           content = await page.content()
           await browser.close()
           return content
   ```

   **方案 B: 手动逆向瑞数6**（备选）
   - 分析动态生成的 JS 代码
   - 还原请求签名算法
   - 模拟 Cookie 生成过程
   - 工作量: 3-5 天，且需要持续维护

   **方案 C: 第三方反反爬服务**（兜底）
   - 使用付费代理服务（如 Bright Data、Oxylabs）
   - 优点: 稳定可靠
   - 缺点: 持续成本

3. **列表页抓取**:
   - URL: `https://www.nmpa.gov.cn/yaopin/ypfgwj/index.html`
   - 分页: `index_2.html`, `index_3.html`, ..., `index_27.html`
   - 等待页面渲染完成
   - 提取列表项

4. **详情页抓取**:
   - URL 格式: `/yaopin/ypfgwj/20260527xxxx.html`
   - 提取正文、附件链接

### 风险评估

| 风险项 | 等级 | 说明 |
|--------|------|------|
| 反爬机制 | **高** | 瑞数6 是商业级反爬产品，绕过难度大 |
| 反爬升级 | **高** | 瑞数6 持续更新，绕过方案可能失效 |
| 页面结构变化 | 中 | 政府网站结构相对稳定 |
| 访问频率限制 | 中 | 可能存在 IP 封禁风险 |
| 技术实现难度 | **高** | 需要深入理解瑞数6 机制 |
| 维护成本 | **高** | 反爬绕过方案需要持续维护 |

**风险等级**: 🔴 高

### 建议

- **是否建议进入 Phase 1**: ⚠️ **有条件进入**
- **条件**:
  1. 先验证 Playwright stealth 模式是否能绕过瑞数6
  2. 如果 stealth 模式失败，评估逆向工程工作量
  3. 如果逆向工程难度过大，考虑使用第三方反反爬服务
  4. 建立反爬监控机制，及时发现绕过失效

- **理由**: 
  - NMPA 是药品监管核心来源，业务价值极高（⭐⭐⭐⭐⭐）
  - 但瑞数6反爬机制是主要障碍
  - 建议先做技术预研（1天），确认可行性后再正式开发

### 预估开发工作量

| 任务 | 工作量 | 说明 |
|------|--------|------|
| 技术预研 | 1 天 | 验证 Playwright stealth 是否可绕过瑞数6 |
| 反爬绕过方案 | 1-3 天 | 取决于瑞数6 绕过难度 |
| 列表页爬虫 | 1 天 | Playwright + 选择器适配 |
| 详情页爬虫 | 0.5 天 | 正文 + 附件提取 |
| 测试 & 调试 | 1 天 | 边界情况 + 反爬监控 |
| **合计** | **4.5-6.5 天** | 含技术预研 |
