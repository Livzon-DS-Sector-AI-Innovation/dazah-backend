# 识林（ShilinX）数据源可行性评估报告

**评估日期**: 2026-06-10  
**评估目标**: 评估识林作为法规情报平台数据源的可行性  

---

## 一、识林平台概述

### 1.1 平台定位

识林（ShilinX）是一个**医药法规聚合与智能检索平台**，定位为"医药 AI 智能问答平台"。

**核心功能**:
- 法规文件智能检索（NMPA、CDE、FDA、EMA 等）
- AI 智能问答（基于 RAG 技术）
- 标准查询（中国药典、USP、EP 等）
- 药品审评数据查询
- 法规更新追踪

**数据来源**:
- 国家药品监督管理局（NMPA）
- 药品审评中心（CDE）
- 美国 FDA
- 欧洲 EMA
- 其他国际监管机构

**产品形态**:
- 网页版：https://www.shilinx.com
- APP：iOS / Android
- 微信小程序：识林法规检索

### 2.5 程序化访问

**调研结果**: ⚠️ **技术上可行，但合规性存疑**

**技术方案**:

#### 方案 A：使用官方 API（如果有）

**前提**:
- 需要企业认证
- 需要付费授权
- 需要签订合作协议

**优点**:
- ✅ 合规合法
- ✅ 稳定可靠
- ✅ 数据质量高

**缺点**:
- ❌ 成本未知（可能较高）
- ❌ 需要商务洽谈
- ❌ 可能有使用限制

#### 方案 B：网页爬虫

**技术可行性**:
- 识林使用现代前端框架（Vue/React）
- 可能有反爬机制（需要测试）
- 需要处理登录和认证

**合规性风险**:
- ⚠️ 可能违反用户协议
- ⚠️ 可能涉及法律风险
- ⚠️ 可能被封锁 IP

**结论**:
- ✅ 技术上可行（如果有 API 或使用爬虫）
- ❌ 爬虫方案合规性存疑
- ⚠️ 推荐使用官方 API（如果可获取）

## 三、用户协议与合规性分析

### 3.1 用户协议关键条款

**调研结果**: ⚠️ **需要详细阅读用户协议**

**常见限制条款**（推测）:
- 禁止未经授权的自动化访问
- 禁止批量下载或爬取
- 禁止商业用途（个人会员）
- 数据仅限个人学习使用

**企业版/机构版**:
- 可能允许数据集成
- 可能允许 API 调用
- 需要签订专门的合作协议

**结论**:
- ⚠️ 个人会员可能禁止程序化访问
- ✅ 企业版可能允许（需要确认）
- ❌ 需要详细阅读用户协议

---

### 3.2 法律风险

**风险点**:
1. **版权风险**: 法规文件本身无版权，但识林的整理、标注可能有版权
2. **合同风险**: 违反用户协议可能导致账号封禁或法律诉讼
3. **技术风险**: 爬虫可能触发反爬机制，导致 IP 封禁

**合规建议**:
- ✅ 使用官方 API（如果有）
- ✅ 签订企业合作协议
- ❌ 避免使用爬虫（除非获得授权）

## 四、商业可行性评估

### 4.1 成本分析

**识林企业版成本**（推测）:
- 年费：可能 5-20 万/年（企业规模）
- API 调用费：可能按次计费或包年
- 定制开发费：可能需要额外付费

**自建爬虫成本**:
- 开发成本：2-4 周（含反爬处理）
- 维护成本：持续（应对反爬升级）
- 法律风险成本：不确定

**第三方数据 API 成本**:
- 摩熵数科、医药魔方等：可能 10-50 万/年
- 数据质量高，合规合法

**结论**:
- ⚠️ 识林企业版成本未知，需要商务洽谈
- ❌ 爬虫方案有法律风险
- ✅ 第三方 API 合规但成本可能更高

## 六、推荐方案

### 6.1 推荐等级

**B. 推荐半自动接入**

### 6.2 推荐理由

#### 为什么不是 A（推荐直接接入）？

1. **API 未公开**: 识林虽然有 API，但未公开文档，需要商务洽谈
2. **成本未知**: 企业版价格不明确，可能较高
3. **授权不确定**: 是否能获得数据集成授权不确定

#### 为什么不是 C（不推荐）？

1. **数据质量高**: 识林提供结构化、全面的法规数据
2. **技术优势**: 相比直接采集 CDE/NMPA，技术难度低
3. **合规性优势**: 如果能获得授权，合规性远优于爬虫

#### 为什么是 B（推荐半自动接入）？

**半自动接入方案**:

1. **第一阶段：商务洽谈**（1-2 周）
   - 联系识林商务团队
   - 了解 API 接口和价格
   - 确认数据集成授权
   - 签订合作协议

2. **第二阶段：技术验证**（1 周）
   - 获取 API 文档和测试账号
   - 测试 API 功能和限制
   - 评估数据质量
   - 验证集成方案

3. **第三阶段：正式接入**（2-4 周）
   - 开发 API 集成代码
   - 实现数据同步机制
   - 测试和上线

**如果商务洽谈失败**：
- 降级为方案 C（不推荐）
- 或继续使用 CDE/NMPA 直接采集（需要解决反爬问题）
